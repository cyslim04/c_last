# 登录后首页与顶部导航重构方案

## 摘要
- 登录后默认入口改为新首页 `"/home"`，首页承担品牌介绍、平台价值说明、流程结构介绍和角色导流；原“控制台”内容并入首页，不再保留独立一级页面。
- 全局壳层从“左侧固定侧栏 + 右侧内容区”改为“顶部统一导航 + 全宽内容区”，所有业务页继续共用同一套 `AppShell`。
- 首页动画走“克制高级”路线，只做滚动触发的淡入、上移、分段 reveal 和轻量 sticky 效果，不做强视差或夸张转场。

## 关键改动
- 路由层新增认证后首页 `"/home"`，`"/"` 重定向到 `"/home"`，旧 `"/dashboard"` 改为兼容重定向到 `"/home"`，登录成功跳转目标同步改为 `"/home"`。
- `AppShell` 重构为统一顶部栏：左侧放品牌与当前角色，中间放主导航，右侧保留钱包连接和会话菜单；桌面端彻底移除固定左侧栏、内容区左边距和依赖侧栏宽度的顶部定位逻辑。
- 顶部导航不再保留当前侧栏的编号块和大卡片样式，改为简洁的 tab / pill 式入口；首页始终排第一位。
- 管理员导航顺序固定为“首页 / 成员管理 / 新增项目 / 项目管理 / 流程总览 / 审计管理 / 个人中心”；开发者为“首页 / 工时记录 / 交付物 / 流程总览 / 个人中心”；客户为“首页 / 客户确认 / 流程总览 / 个人中心”。
- 移动端不再从左侧抽屉展开导航，改为顶部菜单按钮展开下拉式导航面板，保证信息架构与桌面一致。
- 顶栏不再把路由标题当主视觉标题，各业务页标题保留在各自内容区；首页则使用更完整的首屏介绍。
- 新首页首屏承接品牌介绍与平台一句话说明，展示角色主行动按钮、当前系统摘要和一个简洁的状态视觉区，保持大留白、细边框、低饱和中性色和少量强调色。
- 首页第二屏固定为“平台如何协作”，直接复用现有 `api.getProcessFlow()` 的 `globalOverview` 做高层结构展示，只展示结构关系与闭环逻辑，不复用运行态 badge。
- 首页第三屏固定为“当前系统状态”，复用原 `api.getDashboard()` 的指标、待推进事项和摘要逻辑，但压缩成更适合首页的 3 到 4 个关键卡片与 1 个待办摘要区。
- 首页第四屏固定为“按角色开始操作”，沿用原控制台快捷入口语义，但改成首页导流文案；管理员优先导向审计/项目/成员/流程，开发者导向工时/交付/流程，客户导向确认/流程。
- 原 `DashboardView` 的数据装配逻辑迁入首页；旧控制台模板不再保留为独立 UI，旧路径只承担兼容跳转职责。
- 首页滚动动画使用 `IntersectionObserver` 驱动 class 切换，动画范围限定在章节进入、指标卡片错位显现和结构区块渐进展示，避免引入实验性浏览器特性。
- 现有 light/dark 主题接口保持不变，只扩展顶部导航与首页章节样式，不重做整套主题系统。
- `main.css` 需要同时新增顶部导航与首页章节规则，并清理侧栏遗留布局、hover 位移和旧导航语义样式，避免视觉和 DOM 结构脱节。
- 本次变更完成后补 1 篇中文日志到 `docs/logs/2026-W19/2026-05-08-03-顶部导航与首页重构.md`；当前日志总数为 28，新增后变 29，不触发“每 5 篇日志更新 README.md”的阈值，因此 README 本轮不更新。

## 接口与兼容
- 前端公开路由约定会新增 `"/home"`，并让 `"/"` 与 `"/dashboard"` 都落到 `"/home"`。
- 不新增后端接口，不修改认证、钱包、流程图或仪表盘 API 返回结构。
- 钱包菜单、用户菜单、主题切换、角色权限拦截和现有业务页路由权限规则保持不变。

## 测试与验收
- 在 `frontend/` 运行 `npm run build`，确认首页、导航和路由改造可通过构建。
- 用管理员、开发者、客户三类账号分别登录，确认默认都进入首页，且顶部导航只显示各自允许访问的入口。
- 直接访问旧 `"/dashboard"` 时应自动进入 `"/home"`，登录页提交后也应落到 `"/home"`。
- 桌面端不应再出现左侧栏残留，移动端导航应从顶部展开，钱包连接、用户菜单和主题切换必须继续可用。
- 首页向下滚动时章节应按顺序 reveal，无明显卡顿；系统开启“减少动态效果”后，动画应被抑制。
- 从首页进入审计管理、成员管理、项目管理、流程总览、个人中心等页面，确认内容不被 sticky 顶栏遮挡，页面可达且返回导航状态正确。

## 假设与默认值
- 默认不做登录前公开首页，登录页继续只承担登录职责。
- 默认使用统一全局顶栏，不为首页单独做另一套沉浸式头部结构。
- 默认不改后端、不新增接口；首页全部基于现有 `getDashboard()` 与 `getProcessFlow()` 数据拼装。
- 默认保留现有主题切换与配色 token，只做版式、层级和动效升级，不引入新的重视觉风格分支。
