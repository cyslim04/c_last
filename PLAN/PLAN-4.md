# 前端配色极简化整改方案

## Summary
- 目标是把现有偏“黑金/黑紫/AI 感”的视觉，收敛为“双主题保留、黑白灰为主、接近扁平化”的简洁方案。
- 配色方向采用“纯灰极简”，管理员/开发者/客户不再用独立主色驱动界面，只保留语义状态色。
- 本次不改页面结构、不改交互流程，主要通过全局主题变量和少量兼容性样式调整完成。

## Key Changes
- 在 `frontend/src/styles/main.css` 重做两套主题变量：
  - 暗色主题改为中性深灰背景 + 灰阶面板 + 单一中性色强调。
  - 亮色主题改为白灰背景 + 浅灰面板 + 单一中性色强调。
  - 保留 `light` / `dark` 主题切换接口，不改 `theme` store 的行为。
- 去掉高“AI 感”视觉来源：
  - 删除或显著弱化 `body` 背景里的多层 radial/linear gradient。
  - 去掉大面积角色色渐变、发光描边、彩色阴影、sheen 扫光效果。
  - 将毛玻璃 `backdrop-filter` 和重阴影统一降级为轻阴影或纯边框层次；若某些区域可直接平面化，则优先平面化。
- 统一角色视觉策略：
  - 保留现有 `shell-tone-*`、`tone-*` 类名和 `--role-rgb` 机制以避免改动业务组件逻辑。
  - 但这些角色 tone 在壳层、导航、Hero、卡片、按钮选中态上统一映射到同一套中性色，不再切换管理员金色、开发者蓝色、客户绿色主视觉。
- 保留必要语义色，但降复杂度：
  - `success / warning / danger` 继续存在，用于状态标签、反馈文案、审批结果等功能性场景。
  - 这些语义色改成低饱和、低发光版本，不再扩散到页面大底、卡片背景或装饰性光效。
- 重点收敛的组件区域：
  - 导航选中态、顶部身份 pill、钱包按钮、用户菜单、PageHero、各类 panel/card、流程节点、状态卡片、登录页入口卡片。
  - 原本依赖角色色渐变的背景块改为灰阶底 + 细边框 + 轻 hover/focus 反馈。

## Public Interfaces
- 无后端 API、路由、Store 对外接口变更。
- 保留 `data-theme="light|dark"`、主题切换按钮、现有类名约定，避免影响现有组件调用方式。

## Test Plan
- 主题验证：
  - 在亮色和暗色下分别检查 `Login`、`Dashboard`、`Members`、`Projects`、`ProcessFlow`、`Profile` 页面。
  - 切换主题后确认背景、面板、导航、按钮、弹层都已使用新灰阶体系。
- 角色验证：
  - 分别以管理员、开发者、客户登录，确认整体主视觉不再因角色切换而变色。
  - 确认角色信息仍可通过文本、标签内容和信息结构识别，不依赖颜色区分。
- 语义验证：
  - 状态标签、成功/失败/警告提示、审批结果、连接状态仍有清晰可辨的语义色。
  - 文本对比度、边框可见性、hover/focus 态在双主题下可读。
- 交互与响应式验证：
  - 检查侧边栏、顶部栏、钱包弹层、用户菜单、移动端抽屉在去除毛玻璃和彩色光效后层级仍清楚。
  - 确认没有因背景扁平化导致的按钮边界丢失、卡片混层、表单输入框不明显问题。

## Assumptions
- 本次只做前端视觉整改，不改文案、布局、业务逻辑和接口。
- 双主题继续保留，且两套主题都要完成，不做“只修亮色、暗色降级”处理。
- 角色主视觉统一为中性色；仅功能性状态保留弱语义色。
- 优先通过 `main.css` 和现有样式钩子完成整改，尽量不改组件结构；只有在个别组件强绑定旧视觉时，才做最小模板/class 调整。
