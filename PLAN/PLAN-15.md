# 流程总览改为泳道流程图方案

## Summary
- 立体叠放卡片方案整体废弃，三视图统一改为“泳道流程图”视觉语言。
- 方向锁定为：三条角色泳道 `管理员 / 开发者 / 客户`，流程步骤沿时间顺序横向推进，保持当前灰阶极简风格。
- 动效强度采用“中动效”：保留节点切换、路径扫光、展开过渡和当前步骤聚焦，但不做自动巡航或大幅位移动画。
- `test/process-flow-prototype/` 与正式 `/process-flow` 同步改造，避免原型和系统再次分叉。

## Key Changes
- 高层流程视图：
  - 将现有 `process-map-board` 的立体卡片地图替换为横向泳道图。
  - 三条泳道固定按 `管理员 -> 开发者 -> 客户` 纵向排列，节点按实际流程顺序横向布局：`项目创建 -> 钱包准备 -> 阶段登记 -> 客户确认 -> 管理员阶段审计 -> 最终总审计`。
  - 节点不再用悬浮大卡，而改为“站点节点 + 紧凑信息块”；选中节点在本泳道内放大并展开摘要，右侧详情面板保留。
  - 回环关系继续显示为从 `管理员阶段审计` 返回 `阶段登记` 的回流线，但线条风格改为平面路径，不再有 3D 透视感。
  - 中动效具体表现为：当前路径轻扫光、选中节点平滑放大、泳道 hover 高亮、详情切换淡入。

- 阶段循环视图：
  - 保留顶部角色条，但视觉改成泳道头部摘要条，与高层流程共享同一套标签和状态样式。
  - 当前阶段改为“当前阶段泳道板”：三条角色子泳道展示开发者、客户、管理员在本阶段各自所处动作、状态、阻塞和下一步。
  - 历史阶段不再是主舞台侧轨卡片，而改为下方按阶段号倒序排列的折叠泳道行；展开后显示该阶段三角色状态和门禁说明。
  - 当前阶段默认展开且始终位于最上方，历史阶段默认折叠，不再出现大面积悬浮卡堆叠。

- 当前快照视图：
  - 取消三张并列摘要卡，改为“门禁总览板”。
  - 上半区显示当前阶段、最终总审计、当前阻塞三段式门禁条；下半区显示三角色责任状态矩阵。
  - 阻塞原因直接挂在对应角色或门禁节点下，避免用户在三张卡之间来回比对。
  - 视觉语言继续沿用泳道和节点，不再出现独立卡片宇宙。

- 原型与正式页同步：
  - `test/process-flow-prototype/` 同步重做为泳道版本，保留三视图切换和 mock 数据。
  - 正式页在 `frontend/src/views/ProcessFlowView.vue` 重构视图模板，`frontend/src/styles/main.css` 重写流程总览相关样式块。
  - 不新增第三方库，全部使用 Vue 状态切换、CSS Grid/Flex、SVG 路径和现有 `StatusTag` 完成。

## Public Interfaces
- 后端 `/api/process-flow` 字段不变，不新增接口。
- 保留现有 `viewMode` 三视图切换、项目切换器、`roleProgress / globalSteps / stageSummaries / finalAuditStatus` 消费方式。
- 允许前端新增仅用于展示的本地派生结构，例如：
  - `laneOrder`
  - `globalLaneSteps`
  - `currentStageLaneRows`
  - `snapshotGateRows`
- 不改路由、Store、权限逻辑和现有状态值语义。

## Test Plan
- 原型验证：
  - 打开 `test/process-flow-prototype/`，确认三视图都已从立体卡片改为泳道语言。
  - 确认高层流程的回环线、当前节点高亮、详情联动和中动效正常。
- 正式页验证：
  - `/process-flow` 默认进入 `高层流程`。
  - 管理员、开发者、客户三种角色下切换项目，泳道节点、当前阶段和门禁状态同步更新。
  - `阶段循环` 中当前阶段默认展开，历史阶段可折叠展开，三角色状态清楚。
  - `当前快照` 中无需阅读三张独立卡，也能直接看出当前阻塞和最终总审计门禁。
- 响应式与构建：
  - 桌面端泳道横向展示，小屏下自动转为纵向压缩布局，不出现横向溢出。
  - 运行 `cd frontend; npm run build`，构建通过。
  - 检查 `test` 原型和正式页都不再残留“立体卡片流程图”文案与对应 3D 样式类。

## Assumptions
- 本次“不喜欢立体卡片形式”针对整个流程总览三视图成立，不保留任何 3D 叠放卡片主视觉。
- 整体配色继续沿用当前灰阶极简体系，不借这次改动重新引入多角色主色。
- 中动效定义为“有状态反馈但不抢阅读中心”，默认不加入自动播放型动画。
- 原型目录继续保留，作为后续流程视觉讨论的唯一对照版本。
