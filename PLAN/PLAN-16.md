# 高层泳道点击态与模块尺寸修正方案

## Summary
- 本次只修正 `高层流程` 这张泳道主图，不改 `阶段循环` 和 `当前快照` 的信息结构。
- 解决三类问题：
  - 点击后主图高亮关系错误
  - 泳道节点尺寸不一致
  - 左侧泳道成员标签与当前点击节点对应错误
- 交互规则锁定为：
  - 点击后只高亮“点击项”
  - 当前步骤只保留弱提示，不再与点击项并列强高亮
  - 所有泳道节点固定尺寸，超出文字截断，完整信息只看右侧详情

## Key Changes
- 高亮逻辑修正：
  - `selectedGlobalStep` 成为高层泳道图唯一主焦点来源。
  - 左侧泳道标签、泳道轨道、节点、关联连线都只跟随“被点击节点”高亮。
  - `isCurrent` 不再参与左侧标签 `active` 判定，也不再触发整条泳道强高亮。
  - `isCurrent` 仅保留节点自身的弱提示样式，例如细环、轻边框或小状态标记，层级明显低于选中态。
  - 当没有手动点击切换时，默认用当前步骤初始化选中态；一旦用户点击其他节点，主图焦点完全切到该节点。

- 连线动画修正：
  - 只让与选中节点相连的前后链路进入 `link-active`。
  - `link-active` 增强为更高对比度、更粗线宽和轻微发光；其余链路退为低对比背景线。
  - 回环线同样遵守“只跟随点击项高亮”的规则，不再因为当前步骤在别处而误亮。
  - 移除任何会让非选中链路看起来像主路径的动画强度，避免“点了但像没点对”的错觉。

- 节点尺寸统一：
  - 高层泳道节点统一固定宽高，不再允许标题长度把卡片撑高。
  - 推荐实现为固定宽度 + 固定最小/最大高度一致，标题限制 2 行，其他辅助文本不在节点内展示。
  - 节点内部只保留：序号、标题、状态标签。
  - 删除节点中的额外摘要段落；完整说明统一放到右侧详情面板。
  - 选中节点不再通过放大改变尺寸，只通过边框、阴影、背景和连线高亮表达焦点，保证所有节点网格对齐。

- 左侧成员标签修正：
  - 左侧泳道标签卡的 `active` 只在该泳道包含当前选中节点时触发。
  - 取消“选中步骤 OR 当前步骤”并列生效的旧逻辑，杜绝双泳道同时高亮。
  - 标签卡只保留短角色名和泳道标题，不再承载长说明文本，减少误读。

- 原型同步：
  - `test/process-flow-prototype/` 同步应用同一套点击态和固定尺寸规则。
  - 原型与正式页保持一致的高亮判定、连线高亮、节点尺寸和文本截断策略。

## Public Interfaces
- 后端 `/api/process-flow` 无字段变化。
- 路由、项目切换器、`viewMode` 三视图切换逻辑不变。
- 前端仅调整本地派生展示逻辑与样式：
  - 高层泳道的 `active/focused/selected` 判定
  - 连线状态类映射
  - 节点尺寸与截断规则

## Test Plan
- 高层泳道点击验证：
  - 默认进入页面时，当前步骤节点高亮，只有对应泳道标签高亮。
  - 点击其他泳道节点后：
    - 被点击节点高亮
    - 左侧对应泳道标签切换到正确角色
    - 关联连线切换为高亮
    - 原当前步骤不再保持强高亮
  - 点击管理员、开发者、客户泳道节点各一次，确认不会出现双泳道同时高亮。

- 尺寸与排版验证：
  - 所有高层泳道节点宽高一致。
  - 长标题不会把单个节点撑高，只会截断或换行到既定上限。
  - 右侧详情仍能显示完整标题、解锁条件和说明。
  - 桌面端节点横向排列整齐；小屏下可以转单列或横向滚动，但不能出现尺寸错乱。

- 原型与构建验证：
  - `test/process-flow-prototype/` 页面点击态与正式页一致。
  - 运行 `cd frontend; npm run build`，构建通过。
  - 检查正式页与原型中不再存在“泳道标签因当前步骤残留而误亮”的情况。

## Assumptions
- 本次修复聚焦 `高层流程` 主图，不扩展修改 `阶段循环` 和 `当前快照` 的结构。
- “高亮显示”定义为：节点、泳道标签、泳道轨道、关联连线四层同时响应点击。
- 固定尺寸优先于节点内文本完整显示，详细信息统一交给右侧详情面板承担。
- 当前步骤的系统语义仍保留，但视觉上让位于用户当前点击选择。
