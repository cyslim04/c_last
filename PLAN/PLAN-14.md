# 流程阶段主舞台与日志序号命名整改方案

## Summary
- 将 `/process-flow` 的 `阶段循环` 视图改为“当前阶段大面板 + 其余阶段侧轨小卡”的布局。
- 当前阶段继续展示完整门禁、下一步、阻塞、角色状态和交付信息；其余阶段按阶段号顺序缩小排列，并加入轻量进入、hover、选中展开动画。
- 版本日志文件统一改为 `YYYY-MM-DD-序号-中文主题.md`，同一天从 `01` 开始递增，便于区分同日多次更新。

## Key Changes
- 阶段循环布局：
  - 保留顶部紧凑角色条。
  - 将当前阶段面板作为主舞台，占据主要宽度；桌面端左主右侧轨，小屏端上主下侧轨。
  - 新增“阶段侧轨”，展示除当前阶段外的所有阶段，按 `stageNo` 升序排列。
  - 侧轨小卡只展示阶段号、阶段状态、责任方、摘要和关键指标；点击后在卡内展开简短详情，不抢当前阶段视觉中心。
  - 当前阶段主面板加入更明显的进入动画、轻微抬升和当前状态脉冲边框；侧轨卡加入 stagger 进入、hover 抬升、展开过渡。

- 前端实现：
  - 在 `frontend/src/views/ProcessFlowView.vue` 中新增 `orderedStages`、`railStages` 之类的 computed，统一按阶段号排序并排除当前阶段。
  - 调整 `loop` 视图模板，把现有“当前阶段 section + 历史阶段 section”改为一个主舞台布局容器。
  - 复用现有 `toggleStage` / `isStageExpanded` 逻辑，侧轨阶段仍可展开查看。
  - 不改 `/api/process-flow` 字段，不改路由、项目切换、角色权限或后端逻辑。

- 样式实现：
  - 在 `frontend/src/styles/main.css` 中新增/调整 `process-stage-theater`、`process-stage-main`、`process-stage-rail`、`process-stage-mini-card` 等样式。
  - 桌面端使用 `grid-template-columns: minmax(0, 1.35fr) minmax(280px, 0.65fr)`；小屏端自动变为单列。
  - 动画只使用 CSS transition/keyframes，不引入第三方库，保持当前极简灰阶风格和小圆角策略。
  - 侧轨高度过长时允许内部自然纵向排列，不做复杂滚动容器，避免移动端阅读困难。

- 日志命名：
  - 将 `docs/logs/YYYY-Www/` 下现有日志统一重命名为 `YYYY-MM-DD-序号-中文主题.md`。
  - 同一天按当前文件名排序后编号，例如 `2026-04-27-01-流程总览重做与原型接入.md`、`2026-04-27-02-前端极简配色收敛.md`。
  - 更新 `AGENTS.md` 和 `README.md` 中的日志命名规则与示例。
  - 本次新增日志也使用新格式，例如 `2026-04-28-02-流程阶段主舞台与日志命名优化.md`，因为当天已存在一篇 `2026-04-28` 日志。

## Test Plan
- 运行 `cd frontend; npm run build`，确认 Vue 与样式构建通过。
- 手动检查 `/process-flow` 的 `阶段循环` 视图：
  - 当前阶段占据主面板，信息完整且明显大于其余阶段。
  - 其余阶段按阶段号顺序缩小排列，点击可展开，展开后不破坏整体布局。
  - 管理员、开发者、客户三种角色下项目切换后，当前阶段和侧轨阶段同步刷新。
  - 桌面端为主舞台 + 侧轨，小屏端为主舞台 + 下方小卡列表，无文本溢出。
- 检查 `docs/logs/`：
  - 所有日志文件名均符合 `YYYY-MM-DD-序号-中文主题.md`。
  - 同一天序号从 `01` 开始连续递增。
  - `README.md` 和 `AGENTS.md` 中不再出现旧规则 `YYYY-MM-DD-中文主题.md`。

## Assumptions
- “当前阶段占据大量面板，其余按照顺序缩小排列”仅作用于 `阶段循环` 视图，不改 `高层流程` 和 `当前快照`。
- 阶段顺序按 `stageNo` 升序排列，当前阶段不在侧轨中重复出现。
- 日志序号每天独立递增，不跨周、不跨日期连续编号。
- 旧日志内容作为历史记录保留原文，不批量改写历史日志正文，只重命名文件并更新当前规则文档。
