# 修复管理员成员页侧边栏与快捷登录联动

**摘要**
- 锁定桌面端壳层，消除页面级横向滚动，让左侧侧边栏保持固定宽度且不随内容区移动；移动端继续保留抽屉式侧边栏。
- 管理员新增成员时移除“姓名”输入，只保留“用户名”；后端把 `name` 自动回填为 `username`，并把用户名唯一性校验扩展到所有现有用户。
- 登录页快捷登录从前端硬编码改为后端驱动；新增用户、重置密码、用户主动改密后都同步更新快捷登录密码，登录页退出后即可看到最新可用账号。

**接口与数据变更**
- `POST /api/users` 将 `name` 改为可选字段；当缺失或为空时，服务端使用 `username` 作为 `name` 入库。
- 新增公开接口 `GET /api/auth/quick-accounts`，返回 `items[]`，每项包含 `id`、`role`、`displayName`、`username`、`password`、`description`；仅返回 `active` 且存在快捷登录密码的账号。
- `users` 表新增 `quick_login_password` 可空字段；在种子写入、管理员新增成员、管理员重置密码、用户自己修改密码时同步维护该字段。
- 迁移采用增量方式，不提升 `seed_version` 重建整库；已有内置演示账号做一次性回填，无法恢复明文密码的旧自定义账号默认不出现在快捷登录中，直到下一次重置或改密。

**实现变更**
- 调整桌面端壳层布局为“固定宽度侧边栏 + 受限宽度主内容”，并把滚动责任收敛到内容容器，消除 `margin-left + auto width` 导致的整体横向滚动。
- 管理员成员新增表单删除“姓名”字段，创建成功后的列表、详情、顶部身份区仍继续使用 `name` 字段显示，但新建成员初始 `name=username`；成员编辑页和个人资料页继续保留姓名编辑能力。
- 用户名查重改为全局校验 `admin/developer/client` 所有用户，重复时返回明确错误；前端在创建失败时直接展示该错误。
- 登录页保留现有“三个下拉组”交互，但数据源改为接口拉取并按 `admin / developer / client` 分组显示；`client` 组标题仍显示“客户”。
- 快捷登录描述默认使用 `bio`，若为空则回退为 `company`，再为空则回退为“角色演示账号”；停用或删除账号后，下次进入登录页不再展示该账号。
- 同步更新 README、API 文档、数据库文档和本次版本归档，明确新增公开接口与 `quick_login_password` 字段的演示用途。

**测试计划**
- 桌面端打开成员管理页时，浏览器不再出现页面级横向滚动条；纵向滚动只发生在主内容区，左侧侧边栏宽度与位置保持不变。
- 移动端宽度下，侧边栏仍按抽屉模式开关，不引入新的遮挡或错位。
- 管理员新增一个全新用户名的开发者或客户时可以成功创建，成员详情和顶部显示的初始名称等于用户名。
- 管理员新增与任意已有用户同名用户名时会被后端拒绝，前端显示“用户名已存在”。
- 新增成员后退出到登录页，新的快捷登录项会出现在对应角色分组中；刷新页面后仍存在。
- 管理员重置成员密码、成员自己修改密码后，登录页快捷登录使用的新密码仍能成功填充并登录。
- 停用或删除成员后，该账号不会继续出现在登录页快捷登录中。
- 现有默认演示账号 `admin / dev / dev2 / client / client2` 仍可通过快捷登录正常进入系统。

**假设与默认项**
- 仅“新增成员”流程去掉姓名输入；系统其他地方仍保留并兼容 `name` 字段。
- 登录页公开展示快捷登录密码是毕业设计演示需求的一部分，本次按演示环境处理，不做额外安全收敛。
- 如果当前数据库里存在历史自定义用户且无法从现有哈希推回演示密码，这些用户不会被强行伪造快捷登录，统一通过后续重置密码进入快捷登录列表。
