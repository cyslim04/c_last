# 品牌级科技平台·王牌展示版重设计计划

## 摘要
- 将项目整体定位从“暗色管理后台”升级为“品牌级科技平台”，核心目标是做到：第一眼高级、第二眼清晰、第三眼能讲完整逻辑。
- 采用“全系统统一重构 + 纳入答辩演示模式”的方案，保留现有业务流程与接口能力，不先改业务规则，优先重塑视觉语言、页面结构、交互节奏和演示叙事。
- 设计重点不是单纯加更多特效，而是建立一套统一的体验系统：角色主题、页面主视觉、状态动画、流程叙事、异常反馈、演示导览全部同一语法。
- 主舞台页面明确为：登录页、控制台、流程总览、审计页；其余页面统一收敛到同一设计系统内，避免“几页很强、其他页掉线”。

## 设计与体验体系
- 视觉方向固定为“品牌级科技平台”，底色保持深蓝石墨系，但弱化廉价霓虹感，改为更高级的冷黑、钛灰、蓝青主色，辅以少量金色强调管理控制感。
- 角色主题统一映射：管理员为琥珀金、开发者为冷青蓝、客户为翠绿色；主题仅影响强调色、状态光效、关键按钮和顶部上下文，不改变整体品牌一致性。
- 排版体系升级为“展示级标题 + 稳定中文正文 + 哈希等技术信息用等宽字体”，标题用于建立舞台感，正文用于保证信息可读，链上信息维持技术质感。
- 材质语义固定三种：主舞台面板用半透明硬质玻璃，流程/链路模块用发光描边板，表单和详情页用更克制的雾面面板；避免全站只有一种玻璃卡片。
- 动效体系固定四类：页面转场、状态跃迁、数字增长、链路流动；禁止页面到处随机动，所有动画都必须服务于“当前在发生什么”。
- 增加全局“低动效模式”或自动遵循 `prefers-reduced-motion`，重设计版本可以炫，但不能牺牲可用性与演示稳定性。
- 全局壳层重做为“品牌框架”：左侧导航保留固定，但强化为品牌分区导航；顶部不只是身份区，而是“当前角色 + 当前项目 + 当前阶段 + 演示模式入口”的上下文条。
- 页面头部统一引入 `Page Hero` 结构，每页必须有一个能概括“这页在讲什么”的主视觉区，不再直接从卡片列表开始。

## 前端架构与接口策略
- 第一阶段不新增业务后端接口，优先复用现有 `dashboard / process-flow / audit / users / projects / worklogs / deliverables / confirmations / quick-accounts` 数据。
- 前端新增统一的展示层 ViewModel，将现有 API 返回映射成“舞台状态、流程状态、证据状态、演示文案、角色主题”五类 UI 数据，避免每页自行拼装数据。
- 新增前端级接口/类型约定：`RoleTheme`、`PageHeroState`、`StageStatusVisual`、`PresentationScene`、`PresentationScriptStep`，作为全站视觉和演示模式的统一语言。
- 新增全局 `presentation mode` 状态管理，负责答辩模式开关、当前演示脚本、当前高亮区域、目标角色、自动跳转目标页和导览节奏。
- 新增全局设计 token 层，统一颜色、阴影、面板层级、描边、光效强度、动效时长、状态颜色，不再让页面自己决定表现方式。
- 所有状态值统一映射为视觉等级：`pending / running / completed / failed / blocked / inactive`，并绑定固定色彩、图标、动效和语义文案。
- 如果后续发现个别页面需要额外聚合数据，新增接口也只允许做只读聚合，不改现有业务写入流程；本次计划默认不依赖新的后端业务能力。

## 关键页面改造
- 登录页改造成“品牌入口舞台”，左侧为平台叙事主视觉与实时演示亮点，右侧为登录与角色切换；快捷登录不再只是下拉，而是更像“角色入口选择器”。
- 登录页加入“答辩导览入口”，用户可以选择普通登录或进入导览模式；导览模式从这里启动整场演示，而不是散落在系统内部。
- 控制台重构为“品牌总览台”，首页中心只保留一个主叙事区：当前项目、当前阶段、当前闭环进度和下一步动作；指标卡降级为辅助，不再平铺抢视觉。
- 控制台新增“闭环进度环”或“项目健康度主组件”，把工时、交付、确认、阶段审计、总审计统一压缩成一个高识别度主图形。
- 控制台的待推进事项改成“行动队列”，按角色职责和紧急程度排布，并加明显的状态切换动画，强调系统正在推进。
- 流程总览页升级为全站王牌页面，采用“品牌级链路地图”而不是普通信息列表；左侧为角色泳道，中间为阶段路径，右侧为当前阶段与最终审计摘要。
- 流程总览中的每个阶段节点固定包含：阶段编号、当前状态、主要证据、责任角色、通过条件；当前节点具有明显呼吸光和路径流动效果。
- 流程总览新增时间维度切换，可以在“高层流程 / 阶段循环 / 当前快照”三种视图之间切换，既能讲总逻辑，也能讲当前实例。
- 审计页重构为“审计剧场”，顶部展示当前待审项目与审计影响概览，中部用左右分栏对比“提交证据”与“审计结论”，底部展示历史留痕。
- 审计动作弹窗升级为“决策确认面板”，在点击通过或驳回前明确展示会影响哪些状态，避免它看起来像普通提交按钮。
- 项目管理页改造成“项目编排台”，突出项目卡片、阶段状态、开发者/客户绑定关系，强化“谁在做、做到哪、谁在等谁”的信息结构。
- 成员管理页改造成“角色资源台”，开发者和客户卡片更像人物名片，突出姓名、账号、团队、绑定项目、可用状态和演示入口，而不是单纯列表。
- 开发者工时页和交付物页统一成“工作台视图”，把当前项目、当前阶段、提交入口、历史记录、链上状态放在一个连续流程里，降低来回跳读。
- 客户确认页改造成“确认看板”，按阶段展示“本阶段收到什么、我要确认什么、通过后系统会发生什么”，增强业务闭环感。
- 个人中心和钱包区域改造成“数字身份卡”，头像、角色、账号、钱包网络、最近操作合并为一个更高级的身份面板，减少零散信息块。

## 动效与高级交互
- 页面切换采用统一的共享轴转场：从导航进入页面时，标题、主视觉区、关键状态块具有连续位移动画，不再只是整体淡入。
- 所有主指标使用数字滚动和短暂高亮，不使用突兀跳变；只有变化的数字动，静态数字不做噪声动画。
- 流程节点状态变化时使用固定的状态跃迁动画：待办为弱光、进行中为扫描光、完成为点亮封印、失败为短暂红色震动后落稳。
- 链上回写记录用“链路流动”表现，而不是静态哈希堆叠；哈希仍保留文本，但视觉重心放在“已进入链路、已确认、失败回退”的过程感。
- 悬停反馈统一收敛为轻位移、描边增强、背景亮度上升三级，不允许每类组件各做各的 hover 风格。
- 弹窗、下拉、切换器都采用统一浮层语法，包含出现位移、背景压暗、焦点收拢和关闭回弹，形成品牌一致性。
- 空状态、错误状态、禁用状态必须是设计对象，不允许只显示一行灰字；每类状态都要有视觉结构和下一步建议。
- 加载状态统一使用“骨架 + 微扫描”而不是普通 spinner，品牌感会明显更强，也更适合答辩现场。

## 答辩演示模式
- 演示模式纳入正式设计，不作为临时外挂；入口放在登录页和系统顶部，用户进入后整个系统切换到“导览叙事”状态。
- 演示模式采用脚本驱动，每个脚本步骤固定包含：标题、目标角色、目标页面、聚焦区域、说明文案、推进条件、下一步动作。
- 首版脚本固定为 5 幕：平台总览、开发者登记工时与交付、客户阶段确认、管理员阶段审计、管理员最终总审计。
- 同角色内的页面推进可自动跳转；跨角色步骤通过现有快捷登录能力完成“一键切换身份并进入下一幕”，保证真实演示而不是纯前端假播片。
- 演示模式顶部显示“当前第几幕 / 当前角色 / 这一幕要讲什么”，减少答辩时自己记流程的压力。
- 演示模式支持手动前进、后退、暂停和退出；默认不自动定时翻页，避免现场网络或讲解节奏不同步。
- 演示模式高亮采用聚光遮罩或焦点描边，只突出本幕关键区域，不把整页都点亮导致失焦。
- 演示模式中的文案统一用“讲给评委听”的语气，不复用普通系统提示文案。

## 逻辑完整性与产品打磨
- 所有关键动作补齐“为什么能做 / 为什么不能做”的解释层，例如阶段未关闭、客户尚未确认、总审计条件未满足时，界面要明确告诉用户阻塞原因。
- 每个角色首页都展示“我当前应该做什么”和“完成后系统会流向哪里”，让逻辑不仅完整，而且一眼能理解。
- 所有业务链条补齐历史视图：工时、交付、确认、审计、链上回写都能从当前项目或当前阶段回看，不让证据散落在不同页面。
- 失败支线必须可见，包括客户驳回、阶段审计失败、账号停用、钱包未绑定、无可审记录；这些异常路径会显著提升作品成熟度。
- 所有表单页都加入提交前预览或结果摘要，尤其是交付、确认、审计这类关键动作，提升专业感。
- 所有核心页面都需要答辩友好的文案结构：一句主结论、一段当前解释、一个下一步动作，避免界面只堆数据。

## 测试与验收
- 桌面端验收标准是“进入任一核心页面都有主视觉中心、视觉层次明确、无普通后台感、无页面级横向滚动、无视觉断层”。
- 移动端验收标准是“结构不崩、功能可用、动效简化但不消失”；本次优先级明确偏桌面答辩展示，不追求移动端同等震撼度。
- 角色验收覆盖管理员、开发者、客户三类主流程，确认每个角色都拥有独立气质、明确动作和完整成功/失败路径。
- 演示模式验收覆盖 5 幕脚本串联、角色切换、页面跳转、焦点高亮、退出恢复、快捷登录协同和异常中断恢复。
- 动效验收要求所有高级动画都在主要桌面设备上保持流畅，避免大面积重模糊、重阴影和多层粒子叠加导致卡顿。
- 文案验收要求所有核心状态和空状态都能单独被讲清楚，不依赖开发者现场口头补充。
- 品牌一致性验收要求登录页、控制台、流程总览、审计页和次级业务页之间共享统一的色彩、材质、动效和标题语法。

## 假设与默认项
- 保留当前业务模型、角色权限和接口语义，不在本轮重设计中重写后端流程规则。
- 本轮以“品牌级科技平台”为唯一视觉方向，不走强霓虹赛博风，也不走极度保守的企业后台风。
- 本轮默认保留现有路由结构，演示模式以内嵌导览状态方式实现，不强制新增独立演示站点。
- 本轮默认重点做前端重构和叙事增强；若后续出现单页数据不足，再补只读聚合接口，不提前扩展后端复杂度。
- 本轮默认把“流程总览”和“审计页”作为最强王牌页，其他页面围绕它们服务，不平均分配设计资源。
