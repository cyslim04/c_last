# Trust Work 顶部交互、流程总览与多演示账号优化方案

## Summary
- 以“修正个人入口 UI、重做流程总览主视觉、固定左侧/顶部壳层、补齐多开发者/多客户演示账号”为目标继续优化当前演示系统。
- 流程页改为“全局提示 + 项目动态混合图”结构：全局标准流程缩成提示区，主区域使用双栏混合图展示角色与步骤关系，步骤状态只保留 `completed / running / pending` 三态并强化区分。
- 本轮按你确认的范围处理为“多演示账号”，不把项目模型改成真正的同项目多开发者/多客户主成员结构；同一项目仍维持当前 `developer_id + client_id` 主关联模型。

## Key Changes
### 1. 顶部个人入口与壳层固定
- 调整 `AppShell` 顶部个人区交互，拆成 3 个明确点击目标：
  - 头像：直接进入 `/profile`
  - 用户信息区：仅显示姓名、角色、单位，点击不触发路由
  - 箭头/会话按钮：单独打开会话菜单
- 修正个人入口 UI 问题：
  - 个人入口整体宽度固定，长名字与长邮箱只截断不撑开
  - 会话菜单使用顶层浮层并提升 `z-index`，保证不会被流程切换器、原生下拉、状态标签覆盖
  - 点击空白、`Esc`、路由切换均关闭
- 左侧目录栏与顶部状态栏改为固定壳层：
  - 桌面端左侧栏固定在视口左侧，主内容区独立滚动
  - 顶部状态栏在内容区内固定或粘性吸附，滚动页面时不离开视口
  - 小屏保持可用性：顶部栏继续固定，侧栏改为抽屉式入口，避免占满屏幕

### 2. 流程总览页重构
- 页面结构改为两层：
  - 顶部“全局流程提示”条：仅保留一句摘要、状态图例、`查看标准流程` 按钮
  - 主体“项目动态流程图”：作为默认核心展示
- 将当前原生 `select` 项目切换器替换为自定义暗色项目切换面板，避免浏览器原生白底下拉破坏视觉，也避免与顶部弹层层级冲突。
- 动态流程改为“双栏混合图”：
  - 左栏：角色区，固定展示 `系统管理员 / 当前项目开发者 / 当前项目客户`
  - 右栏：8 个步骤节点的流程图，每个节点使用矩形卡片并用箭头串联
  - 角色区与步骤区用引导线或角色标识说明“谁负责该步骤”，不做严格 UML，用更适合答辩展示的业务流程图表达
- 全局标准流程默认收起，不再大面积平铺 8 步卡片；点击后才展开标准 8 步说明面板。

### 3. 流程状态视觉与判定输出
- 三种状态的视觉规范固定如下：
  - `completed`：高亮亮绿色系，带明显边框/光晕，文本仍保持高对比浅色
  - `running`：高亮青蓝或琥珀色系，与完成态明显区分，带动态强调效果
  - `pending`：灰色低饱和底板与边框，正文文字仍保持清晰，不使用低对比灰字
- 状态标签、流程节点、箭头连接线统一使用同一套状态颜色，不再只改边框。
- 流程节点包含：
  - 步骤名
  - 步骤说明
  - 状态标签
  - 证据摘要
  - 当前责任角色标识
- 后端 `GET /api/process-flow` 保留现有判定规则，同时补充前端渲染所需字段，避免前端自行推导角色归属。

### 4. 多演示账号与角色区分
- 扩展演示种子数据，新增多名开发者和客户账号，保持“简便为主”的毕业设计模式：
  - 管理员：`admin / admin123`
  - 开发者 A：`dev / dev123`
  - 开发者 B：`dev2 / dev123`
  - 客户 A：`client / client123`
  - 客户 B：`client2 / client123`
- 为新增账号补齐可见项目与演示数据，至少保证第二位开发者和第二位客户登录后不是空白视图：
  - 现有完成项目与进行中项目继续给开发者 A / 客户 A
  - 新增 1 个由开发者 B / 客户 B 参与的演示项目，并补齐工时、交付物、部分存证/审批记录
- 登录页改为“按角色分组的快捷登录”：
  - 管理员组、开发者组、客户组分栏展示
  - 每个账号提供“一键登录”按钮，直接调用登录，不再只做“填入账号”
  - 保留手动输入表单作为兜底
- 各账号在登录页和系统内统一展示区分信息：
  - 姓名
  - 角色
  - 所属单位/职责标签
  - 关联项目简述
- 本轮不把项目改成真正的多人主成员模型；“多人区分”体现在多账号、多项目、多职责标签和登录入口上。

## Public APIs / Interfaces
- `GET /api/process-flow` / `GET /api/process-flow?projectId=...` 扩展响应：
  - `projectOptions` 补充 `developerName`、`clientName`
  - `selectedProject` 补充 `developer`、`client` 展示信息，至少包含 `id`、`name`、`username`、`company`
  - `projectSteps` 补充 `ownerRole`，固定为 `admin | developer | client`
  - 可增加 `globalHint`：`{ title, summary }`，供顶部提示条直接展示
- 不新增项目多人协作接口，不修改现有项目创建主字段模型。
- 不做数据库 schema 变更；通过提升种子版本并重建演示数据补齐多账号与多项目。

## Test Plan
- 点击头像直接进入个人中心；点击箭头按钮只展开会话菜单；个人入口在长姓名/长邮箱下不变形、不遮挡。
- 顶部状态栏滚动时固定可见，左侧目录栏在桌面端固定不动；内容区独立滚动不导致导航位移。
- 流程总览页默认以“全局提示 + 动态流程图”为主，不再先看到大面积全局 8 步卡片。
- 项目切换器不再使用原生白底下拉；暗色自定义切换面板在桌面端和小屏端都可用。
- 动态流程图中：
  - `completed` 节点为亮绿色高亮
  - `running` 节点为另一种明显亮色高亮
  - `pending` 节点为灰色底板但文字清晰
  - 箭头、节点、状态标签颜色一致
- 流程图可清楚看出管理员、开发者、客户分别负责哪些步骤。
- `dev` / `dev2` / `client` / `client2` 都能从登录页一键登录，并进入各自有数据的工作台。
- 第二位开发者和第二位客户登录后，控制台、流程总览、交付/确认页都能看到与自己相关的数据，不出现空白演示。
- README 更新本轮功能说明，并在 `docs/` 下新增一份独立版本文档，不覆盖旧文件。

## Assumptions
- “多名开发者或客户时做好区分”按你确认的范围，默认实现为“多演示账号 + 多项目区分”，不升级为“同项目多主成员”模型。
- 流程图按你确认的“`双栏混合图`”实现，不做严格 UML 用例椭圆图。
- 全局流程默认缩成提示区，并提供显式展开入口，不再与动态流程并列大面积展示。
- 登录体验以简便优先，默认提供一键登录；不额外增加复杂安全校验或验证码。
