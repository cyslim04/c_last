# Trust Work 平台

基于区块链演示场景的可信工时、交付物存证与客户确认平台。

## 快速启动

推荐顺序：

1. 先启动后端

```powershell
cd backend
.\run.ps1
```

2. 再启动前端

```bash
cd frontend
npm install
npm run dev
```

3. 如需验证合约，再运行

```bash
cd contracts
npm install
npm run test
```

依赖：

- 后端需要 `gcc` 和 `sqlite3`

默认地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8080`

补充说明：

- `backend/run.ps1` 会在启动前自动停止同路径下已运行的旧后端进程
- `backend/build.ps1` 在编译失败时会直接中断，不再误报 `Build completed`
- 前端默认使用 `http://localhost:8080` 作为 API 地址

## 项目结构

- `frontend/`：Vue 3 + Vite 前端
- `backend/`：C + SQLite 后端服务
- `contracts/`：Solidity 合约与 Hardhat 相关代码
- `docs/reference/`：接口、数据库、演示说明和使用手册
- `docs/logs/`：按周归档的版本更新文档，统一使用 `YYYY-MM-DD-序号-中文主题.md` 命名

## 当前版本能力

### 角色闭环

- 管理员：首页、成员管理、项目管理、审计管理、个人中心
- 开发者：首页、工时记录、交付物、个人中心
- 客户：首页、客户确认、个人中心

### 成员管理

- 新增管理员专用 `成员管理` 页面
- 支持新增开发者和客户账号
- 新增成员时仅需输入用户名，初始展示名默认回填为用户名
- 用户名会与系统内全部已有用户做全局唯一校验
- 支持编辑资料、重置密码、停用/启用和条件删除
- 已停用成员不可登录，也不可再被分配到新项目

### 钱包与 MetaMask

- 连接成功后显示账户缩写和当前网络
- 支持监听账户切换和网络切换
- 支持快速切换 `Sepolia` 和本地 `Hardhat`
- 支持前端断开连接，不影响账号登录态
- 钱包切换后自动回写当前登录用户的钱包绑定
- 钱包菜单改为顶层浮层，小屏下自动退化为底部面板，避免被页面布局遮挡

### 登录与头部体验

- 登录后默认进入首页，不再先落到旧控制台
- 顶部导航改为统一全局栏，品牌、角色身份、业务导航、钱包和会话入口集中在同一层
- 移动端导航改为顶部展开式菜单，不再从左侧抽屉进入
- 顶部左上角改为紧凑身份块，仅显示 `角色 + 姓名 + 登录状态`
- 无真实姓名时自动回退为账号名
- 顶部右上角用户区改为更紧凑的两行信息卡，避免姓名、角色重复显示
- 中文头像 fallback 改为单字展示，避免撑坏圆形头像
- 登录页快捷登录改为 `管理员 / 开发者 / 客户` 三组下拉
- 快捷登录数据由后端公开接口动态提供，不再写死在前端
- 管理员新增成员、管理员重置密码、用户自己修改密码后，快捷登录会自动同步新密码
- 下拉展开时展示姓名、账号、密码与说明
- 选择快捷账号后只填入表单，不自动提交登录

### 视觉与页面结构

- 全站默认使用 `Warm Stone & Paper` 浅色主题，不再提供亮暗色切换
- 页面基底、卡片层级、按钮和状态标签统一到暖石 / 暖墨 / 赤陶配色
- 首页首屏只保留结构说明、角色入口和主 CTA，不再放仪表盘式动态数据卡
- 动态数据下沉到首页中段运行信号区，首页继续保持介绍页职责
- 登录页、首页和主要业务页统一为同一套 Stride 风格 Hero、section 节奏和 footer 收口

### 个人中心

- 查看和修改姓名、邮箱、手机号、公司/团队、个人简介
- 上传头像
- 修改密码
- 退出登录
- 顶部头像可直接进入个人中心，用户名/角色区域单独负责展开会话菜单

### 业务流程

- 管理员创建项目并确定开发者与客户
- 开发者完成首次钱包绑定
- 开发者在当前阶段持续登记工时和交付物
- 开发者对当前阶段继续发起演示链回写
- 客户按阶段查看交付物和同阶段工时历史并做确认
- 管理员在客户通过后执行阶段审计
- 所有阶段关闭后，管理员执行最终总审计

### 项目管理体验

- 管理员创建项目入口与项目管理视图拆分为两个独立页面
- `新增项目` 页面专注于创建项目
- `项目管理` 页面专门用于查看项目状态与筛选
- 项目管理页支持按状态查看：
  - `进行中`
  - `已完成`
  - `争议中`
  - `草稿`
- 项目管理页支持按开发者、客户做组合筛选
- 已完成项目已集中收纳到项目管理页中，不再混在进行中项目里
- 管理员创建项目时，开发者和客户改为暗色自定义成员选择器
- 成员下拉支持点击空白关闭和 `Esc` 关闭
- 成员选项在 Windows 暗色界面下保持可读，不再出现白底白字

### 表单选择器体验

- 开发者工时页和交付物页的“所属项目”已改为统一暗色自定义选择器
- 项目选择与成员选择共享一致的交互方式和视觉表现
- 当前前端源码中的原生 `select` 已完成替换，不再保留同类问题入口

### 流程总览与审计体验

- 首页新增“平台如何协作”“当前系统状态”“按角色开始操作”三段式介绍，用于在进入业务页前先讲清平台结构
- `流程总览` 页面改为“高层流程 + 阶段列表 + 当前阶段 + 最终总审计状态”结构
- 审计页拆分为 `待阶段审计` 与 `待最终审计` 两类队列
- 客户确认页从“待确认交付物”改为“待确认阶段”
- 每个待确认阶段都会展示该阶段工时历史与最新交付
- 审计页继续保留交易记录和后台日志，方便答辩时展示阶段审计与总审计留痕
- 顶部用户菜单与钱包菜单采用顶层弹层，支持点击空白关闭、`Esc` 关闭、路由切换自动关闭
- 左侧目录栏和顶部状态栏固定显示，主内容区独立滚动
- 顶部状态栏现为真正固定，不随页面滚动移动
- 桌面端左侧目录栏固定 300px 宽度，内容区不再把页面整体撑出横向滚动

### 演示数据

- 系统启动时会自动检查数据库版本
- 当数据库为空或种子版本不匹配时，会自动重建完整演示数据
- 默认包含 5 个演示账号、3 个阶段化演示项目、多条工时、多条交付物、阶段确认记录、阶段审计记录和钱包绑定记录
