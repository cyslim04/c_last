# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-28
- 版本说明：流程阶段主舞台与日志命名优化
- 文档类型：版本更新归档

## 本次更新概览

本次更新继续优化 `流程总览` 的阶段循环视图，并统一版本日志命名规则。阶段循环从原来的“当前阶段完整面板 + 历史阶段列表”调整为“当前阶段主舞台 + 其余阶段侧轨小卡”，让当前阶段始终占据视觉中心，同时保留其他阶段的顺序查看能力。日志文件名在日期后增加序号，便于区分同一天内的多次更新。

## 主要更新内容

### 1. 阶段循环主舞台

- 当前阶段继续展示完整的前置条件、下一步动作、阻塞说明、角色状态、交付信息和最终总审计提示
- 当前阶段面板作为主舞台展示，占据页面主要宽度
- 主舞台增加轻微 hover 抬升和当前阶段脉冲边框，突出当前阶段重点

### 2. 阶段侧轨小卡

- 其余阶段按阶段号升序排列在侧轨中
- 小卡展示阶段号、阶段状态、责任方、摘要、工时和交付数量
- 点击侧轨小卡可展开阶段门禁、下一步、阻塞说明和最新交付
- 侧轨小卡加入进入、hover 和展开动画，避免界面静态堆叠

### 3. 日志命名规则调整

- `docs/logs/` 下现有日志统一重命名为 `YYYY-MM-DD-序号-中文主题.md`
- 同一天日志从 `01` 开始按文件名顺序递增
- `AGENTS.md` 与 `README.md` 已同步更新新的日志命名规则
