# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-27
- 版本说明：前端极简配色收敛版
- 文档类型：版本更新归档

## 本次更新概览

本次更新聚焦前端整体视觉风格整改。原有界面在暗色背景、角色色渐变、光晕和毛玻璃的叠加下，呈现出较强的“黑金 / 黑紫 / AI 风”观感。此次调整将主题统一收敛为更克制的灰阶体系，在保留双主题和现有交互结构的前提下，让页面整体更简洁、更稳定，也更适合答辩展示和日常演示。

## 主要更新内容

### 1. 双主题灰阶重构

- 重做 `frontend/src/styles/main.css` 中的暗色与亮色主题变量
- 暗色主题改为中性深灰底色与灰阶面板
- 亮色主题改为白灰主底与浅灰层级面板
- 管理员、开发者、客户三类角色不再使用独立主色驱动整体页面

### 2. 高感知视觉效果收敛

- 移除页面背景中的网格蒙层与大面积彩色氛围渐变
- 关闭卡片扫光、运行态 glow 动画与多处彩色发光阴影
- 将顶部栏、卡片、浮层、钱包按钮、会话入口等区域的毛玻璃效果统一降为普通平面层次
- 按钮、导航激活态、Hero、登录区推荐卡片改为更平整的灰阶反馈

### 3. 交互语义保留

- 保留亮色 / 暗色切换能力，不改动主题 Store 和切换入口
- 保留成功、警告、失败等必要语义色，但整体降低饱和度和装饰性
- 状态标签、连接状态、审批结果仍保持可识别，不再承担整页主视觉角色

## 影响范围

- `frontend/src/styles/main.css` 全局主题变量
- 侧边栏、顶部栏、Hero、登录页、流程总览、弹层、按钮、标签等共享视觉区域
- 不涉及后端接口、前端路由、状态管理和业务流程逻辑

## 验证结果

- 已执行 `cd frontend && npm run build`
- Vite 生产构建通过，未出现样式语法或打包错误
- 角色主视觉已统一为中性灰阶方向，主要页面样式来源已从彩色渐变切换为简洁面板层次
