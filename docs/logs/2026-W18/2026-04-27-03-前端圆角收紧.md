# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-27
- 版本说明：前端圆角收紧版
- 文档类型：版本更新归档

## 本次更新概览

本次更新聚焦前端全局圆角体系收紧。此前界面中的卡片、按钮、状态标签、钱包入口、会话入口和头像等区域普遍使用较大的圆角或胶囊式圆角，整体观感偏柔和。此次调整将全站圆角统一压缩到更克制的范围，让界面更利落，也和前一轮灰阶极简化方向保持一致。

## 主要更新内容

### 1. 全局圆角等级整体下调

- 将 `frontend/src/styles/main.css` 中的圆角变量统一缩小
- `--radius-2xl`、`--radius-xl`、`--radius-lg`、`--radius-md`、`--radius-sm` 均改为更小的取值
- 依赖这些变量的卡片、面板、输入框、弹层、登录模块和流程节点会同步变得更方正

### 2. 写死的大圆角同步收紧

- 状态标签、顶部钱包入口、会话入口不再使用强胶囊式圆角
- 顶栏图标按钮、导航序号块、操作按钮、流程索引块统一改为更小圆角
- 头像卡片、品牌标识块、快捷登录卡片等此前写死的较大圆角也同步缩小

### 3. 视觉一致性处理

- 保留必要的圆形结构，例如环形图这类明确依赖圆形表达的组件
- 其余以矩形卡片、按钮和面板为主的区域统一向“低圆角”方向靠拢
- 调整后页面整体更偏简洁、硬朗，不再出现局部圆角明显偏大的混搭感

## 影响范围

- `frontend/src/styles/main.css` 内的全局圆角变量
- 共享按钮、标签、导航、顶部栏、登录页、流程页、弹层和个人资料相关样式
- 不涉及业务逻辑、接口、路由和状态管理

## 验证结果

- 已执行 `cd frontend && npm run build`
- Vite 生产构建通过，未出现样式语法或打包错误
- 全站主要矩形组件的圆角已明显缩小，圆角风格更加统一
