# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-27
- 版本说明：流程总览重做与原型接入
- 文档类型：版本更新归档

## 本次更新概览

本次更新围绕 `流程总览` 页面进行重做，并按“原型先行 + 正式页接入”的方式同步落地。新增了独立的交互原型目录，用于演示立体卡片地图和阶段展开方式；同时正式页不再停留在原有的纯文字步骤卡片与大角色栏结构，而是重构为更适合讲解流程、观察卡点和查看历史阶段的新版总览。

## 主要更新内容

### 1. 新增独立流程总览原型

- 在仓库根目录新增 `test/process-flow-prototype/`
- 原型包含独立的 `index.html`、`styles.css` 和 `script.js`
- 原型保留 `高层流程 / 阶段循环 / 当前快照` 三视图切换
- 高层流程改为立体卡片流程图，带节点聚焦、连线流动与循环回路
- 阶段循环原型支持顶部紧凑角色条、当前阶段重点卡和历史阶段展开查看

### 2. 正式页重构

- `frontend/src/views/ProcessFlowView.vue` 已按新结构重写
- 高层流程不再是简单文字卡片，改为内联 SVG 连线 + 分层节点卡的互动流程图
- 阶段循环移除旧的大角色栏，改为顶部紧凑角色条
- 当前阶段单独放大显示，历史阶段按倒序折叠排列，点击后内联展开详情
- 当前快照改为门禁看板，突出总审计条件、阻塞原因和当前责任角色

### 3. 流程接口增强

- `/api/process-flow` 在保留旧字段的前提下新增：
  - `roleProgress`
  - `globalSteps[*].actor/state/detail/gate/isCurrent`
  - `stageSummaries[*].isCurrent/phase/currentOwner/requirement/nextAction/blockingReason/roleStates`
  - `finalAuditStatus.blockingReason/blockingStageNo/requirements`
- 这些字段由后端直接生成，前端不再依赖本地猜测流程状态

### 4. 共享样式与状态标签补充

- `frontend/src/styles/main.css` 增加新版流程总览专用样式
- 新增流程图节点、紧凑角色条、当前阶段重点卡、历史阶段折叠体和门禁看板样式
- `frontend/src/components/StatusTag.vue` 补充 `locked` 和 `ready` 状态映射，便于流程总览直接复用
