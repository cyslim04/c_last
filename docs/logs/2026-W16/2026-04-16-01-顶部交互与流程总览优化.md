# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-16
- 版本说明：顶部交互与流程总览优化版
- 文档类型：版本更新归档

## 本次更新概览

本次版本围绕“顶部交互稳定性 + 审计信息层级优化 + 流程可视化”继续完善演示系统。重点解决顶部用户入口和 MetaMask 入口的遮挡问题，同时新增一页面向答辩展示的流程总览页，用于直观看到项目当前运行到哪个步骤。

## 主要更新内容

### 1. 顶部交互层重构

- 钱包菜单和用户菜单改为顶层弹出层
- 浮层挂载到 `body`，避免被顶部布局、父容器或滚动区域遮挡
- 支持点击空白关闭
- 支持按 `Esc` 关闭
- 路由切换后自动关闭
- 小屏下自动退化为更稳定的全宽底部面板

### 2. 用户入口逻辑优化

- 顶部头像点击后直接进入个人中心
- 用户名、角色、箭头区域只负责展开会话菜单
- 会话菜单聚焦退出登录等会话操作
- 不再要求先展开菜单再进入个人中心

### 3. 审计页日志折叠

- 审计页继续默认展示：
  - 最近链上回写
  - 项目闭环状态
- 后台日志改为默认折叠
- 区块头部增加：
  - `查看日志`
  - `收起日志`
- 未展开时显示日志摘要：
  - 日志总数
  - 最近一条时间
  - 最近一条动作摘要

### 4. 流程总览页

- 新增 `/process-flow` 页面
- 页面提供两类展示：
  - 固定标准 8 步流程
  - 当前项目的动态流程状态
- 支持在页面内切换项目
- 项目选择范围遵守当前角色可见权限：
  - 管理员可见全部项目
  - 开发者和客户仅可见与自己相关的项目

### 5. 动态流程状态规则

动态流程的 8 个步骤如下：

1. 管理员创建项目
2. 分配开发者与客户
3. 开发者绑定钱包
4. 开发者登记工时
5. 开发者登记交付物
6. 发起链上存证或演示回写
7. 客户确认或驳回
8. 管理员审计复核结果

每一步统一输出以下三种状态：

- `completed`
- `running`
- `pending`

后端会基于项目、成员、钱包绑定、工时、交付物、存证记录、审批记录和项目状态进行只读聚合计算。

## 新增或调整的接口

- 新增：`GET /api/process-flow`
- 支持：`GET /api/process-flow?projectId=<id>`
- 返回结构包含：
  - `globalSteps`
  - `projectOptions`
  - `selectedProject`
  - `projectSteps`
