# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-20

- 文档类型：版本更新归档

## 本次更新概览

本次修复在“项目管理成员选择器修复”基础上，继续处理开发者侧表单中“所属项目”选择器的同类问题，并对前端所有下拉选择功能做了一次统一自检。最终将原生 `select` 全部替换为统一的暗色自定义选择器，避免 Windows 环境下继续出现白底白字和可读性异常。

## 主要更新内容

### 1. 交付物页所属项目选择器修复

- 修复开发者“登记交付物”页中的“所属项目”下拉
- 原生 `select` 改为统一的暗色自定义选择器
- 收起态直接显示项目名称
- 展开态保持与全站暗色面板一致的视觉风格

### 2. 工时页所属项目选择器同步修复

- 同步修复开发者“登记工时”页中的“所属项目”下拉
- 与交付物页使用同一套选择器组件
- 保持交互一致：
  - 点击空白关闭
  - `Esc` 关闭
  - 暗色高亮 hover 与选中态

### 3. 选择器组件统一

- 扩展现有自定义选择器，使其可同时适配：
  - 成员选择
  - 项目选择
- 组件支持：
  - 自定义 label 输出
  - 可选 meta 展示
  - 通用占位文案

### 4. 类似功能自检结果

- 已检查前端 `frontend/src/` 下所有表单和视图组件
- 自检范围包括：
  - 项目管理页
  - 工时页
  - 交付物页
  - 其他带下拉切换的页面
- 自检后确认：
  - 前端源码内已无原生 `<select>` / `<option>` 残留
  - 现有下拉选择已统一到自定义暗色方案

## 验证结果

- 前端已通过 `npm run build`
- 已确认工时页与交付物页的所属项目选择器已完成替换
- 已确认源码级自检不再存在原生 `select`
