# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-21
- 版本说明：成员管理与快捷登录联动修复版
- 文档类型：版本更新归档

## 本次更新概览

本次版本继续围绕毕业设计答辩演示的稳定性做修复，重点解决管理员成员页与登录页之间的数据脱节问题，同时收紧桌面端壳层布局，避免左侧侧边栏在大页面中出现横向拖动和视觉位移。

## 主要更新内容

### 1. 桌面端壳层固定宽度

- 左侧侧边栏继续保持桌面端固定定位
- 侧边栏宽度明确锁定为 `300px`
- 主内容区宽度改为受限布局，不再因为双栏表单或卡片内容把整页撑出横向滚动
- 页面纵向滚动继续收敛到主内容区，侧边栏与顶部栏保持固定

### 2. 管理员新增成员流程简化

- 成员新增表单移除单独的“姓名”输入
- 管理员新增开发者或客户时，只需填写：
  - 用户名
  - 初始密码
  - 公司 / 团队
  - 邮箱
  - 手机号
  - 个人简介
- 后端在未传入 `name` 时，自动使用 `username` 作为初始展示名
- 编辑成员资料和个人中心仍保留姓名字段，兼容后续人工调整

### 3. 用户名全局唯一与快捷登录持久化

- 成员新增时，用户名会与系统内全部已有用户做全局唯一校验
- 新增 `users.quick_login_password` 字段，作为答辩演示环境的快捷登录密码来源
- 以下场景会同步更新快捷登录密码：
  - 管理员新增成员
  - 管理员重置成员密码
  - 用户自己修改密码
- 旧数据库中的 5 个默认演示账号会按当前真实密码尝试一次性回填快捷登录密码
- 无法恢复明文密码的历史自定义账号，默认不进入快捷登录列表，直到下一次重置或改密

### 4. 登录页快捷登录改为后端驱动

- 新增公开接口：`GET /api/auth/quick-accounts`
- 登录页保留 `管理员 / 开发者 / 客户` 三组下拉结构
- 账号列表改为实时读取后端返回数据，不再硬编码在前端
- 快捷登录项展示内容统一为：
  - 姓名
  - 账号
  - 密码
  - 说明
- 说明字段优先使用：
  - `bio`
  - `company`
  - 角色默认文案
- 停用或删除成员后，账号会自动从登录页快捷登录中消失

## 新增或调整的接口

- 新增：`GET /api/auth/quick-accounts`
- 调整：`POST /api/users`
  - `name` 改为可选
  - 缺省时自动回填为 `username`

## 验证关注点

- 桌面端成员页不再出现页面级横向滚动条
- 新增成员后退出到登录页，可在对应角色分组看到新的快捷登录项
- 管理员重置密码、用户自己修改密码后，登录页快捷密码同步更新
- 停用或删除成员后，该账号不再出现在快捷登录中
