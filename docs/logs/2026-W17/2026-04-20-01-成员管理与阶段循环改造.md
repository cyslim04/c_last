# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-20
- 版本说明：成员管理与阶段循环式流程改造版
- 文档类型：版本更新归档

## 本次更新概览

本次版本完成了两条核心改造：

1. 新增管理员专用“成员管理”页，支持开发者/客户账号的新增、编辑、重置密码、停用/启用与条件删除。
2. 将原先的线性项目流程改造成“项目立项 + 开发准备 + 阶段循环 + 最终总审计”模型，使工时、交付、客户确认和管理员审计都围绕阶段展开。

## 主要更新内容

### 1. 成员管理

- 新增 `/members` 管理员页面
- 支持：
  - 新增开发者/客户
  - 编辑成员资料
  - 管理员重置密码
  - 停用 / 启用
  - 条件删除
- 新增 `users.status`
  - `active`
  - `inactive`
- 停用成员不可登录，也不会出现在新项目分配下拉中
- 已被项目引用的成员不允许删除，只能停用

### 2. 阶段模型

- `worklogs` 与 `deliverables` 新增 `stage_no`
- 后端自动决定当前写入阶段：
  - 若最近阶段已关闭，则自动进入下一阶段
  - 若最近阶段仍开放，则继续写入当前阶段
  - 若最近阶段已通过客户确认、等待管理员阶段审计，则禁止继续登记
- 新增 `project_audits` 表，统一保存：
  - 阶段审计
  - 最终总审计

### 3. 审计与确认流程

- 客户确认页改为“待确认阶段”
- 每个阶段显示：
  - 当前阶段最新交付
  - 当前阶段工时历史
- 管理员审计页改为双队列：
  - `待阶段审计`
  - `待最终审计`
- 新增：
  - `POST /api/audit/stage`
  - `POST /api/audit/final`
- 阶段审计与总审计继续使用演示链回写，新增链上业务类型：
  - `stage_audit`
  - `final_audit`

### 4. 流程总览与摘要

- `/api/process-flow` 改为返回：
  - 高层流程说明
  - `stageSummaries`
  - `currentStage`
  - `finalAuditStatus`
- Dashboard 摘要文案切换为阶段语义
- 审计页项目总览切换为阶段语义

### 5. 演示数据

- 升级种子版本到 `4`
- 重建为 3 个阶段化示例项目：
  - 一个待总审计
  - 一个待阶段审计
  - 一个待客户确认

## 验证结果

- 前端已通过 `npm run build`
- 后端已通过源码编译
- 已验证：
  - `GET /api/users?status=all` 返回成员状态字段
  - 客户账号可读取自身项目工时历史
