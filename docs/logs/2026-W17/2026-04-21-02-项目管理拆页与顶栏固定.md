# Trust Work 平台更新记录

## 版本信息

- 日期：2026-04-21
- 版本说明：管理员项目管理拆页与固定顶栏版
- 文档类型：版本更新归档

## 本次更新概览

本次版本围绕管理员使用体验继续做结构化调整，重点把“创建项目”和“管理项目状态”拆成两个独立页面，并将顶部状态栏改为真正固定显示，便于答辩时长页面滚动下仍保持稳定操作入口。

## 主要更新内容

### 1. 管理员项目入口拆页

- 原 `项目管理` 页面重定义为 `新增项目`
- 新增独立 `项目管理` 页面，专门查看与筛选项目
- 管理员导航改为：
  - 控制台
  - 成员管理
  - 新增项目
  - 项目管理
  - 流程总览
  - 审计管理
  - 个人中心

### 2. 项目管理页分类与筛选

- 项目管理页支持按状态分类查看：
  - `全部`
  - `进行中`
  - `已完成`
  - `争议中`
  - `草稿`
- 支持按开发者、客户继续筛选
- `已完成` 项目已单独收纳，不再混入进行中项目
- 在 `全部` 视图下，项目按：
  - 进行中项目
  - 已完成项目
  - 异常 / 草稿项目
  三段展示

### 3. 新增项目页收敛

- `新增项目` 页面只保留创建逻辑
- 页面文案强调“这里只负责创建，不负责状态管理与筛选”
- 右侧项目列表收缩为最近项目参考，不再承担管理页职责

### 4. 顶部状态栏固定

- 顶部状态栏由 `sticky` 改为真正固定
- 页面滚动时，顶部状态栏不再离开视口
- 主内容区同步增加顶部偏移，避免首屏内容被遮挡
- 移动端仍保持固定状态栏，同时保留侧边栏抽屉逻辑

## 接口与返回说明

- 未新增后端接口
- 继续使用：
  - `GET /api/projects`
  - `POST /api/projects`
  - `GET /api/users?role=developer`
  - `GET /api/users?role=client`
- 项目管理页中的状态分类由前端基于 `GET /api/projects` 返回的 `status` 字段完成

## 验证结果

- 管理员导航中可分别进入 `新增项目` 与 `项目管理`
- `项目管理` 页可按状态、开发者、客户筛选项目
- `已完成` 项目可被单独查看
- 顶部状态栏滚动时保持固定
- 前端已通过 `npm run build`
